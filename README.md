# AES_CRYPTO

Программная реализация алгоритма блочного шифрования AES (Advanced Encryption Standard) на языке Python.

Проект выполнен в учебных целях и не использует сторонние криптографические библиотеки — все преобразования реализованы вручную на основе спецификации FIPS 197.

## Возможности

- Варианты длины ключа: **AES-128**, **AES-192**, **AES-256**
- Режимы работы: **ECB** (Electronic Codebook) и **CBC** (Cipher Block Chaining)
- Дополнение данных по стандарту **PKCS#7**
- Алгоритмическая генерация таблиц S-box, Inv-S-box и Rcon на основе арифметики в поле GF(2⁸)
- Шифрование и расшифрование произвольных файлов через командную строку
- Поддержка передачи ключа напрямую или через файл
- Набор тестов с эталонными векторами из спецификации AES

## Описание алгоритма AES

AES — симметричный блочный шифр с фиксированным размером блока **128 бит** (16 байт). Длина ключа определяет вариант алгоритма и количество раундов шифрования:

| Вариант  | Длина ключа | Раундов |
|----------|-------------|---------|
| AES-128  | 16 байт     | 10      |
| AES-192  | 24 байта    | 12      |
| AES-256  | 32 байта    | 14      |

Входной блок представляется в виде матрицы состояния 4×4 (State), заполняемой по столбцам. Каждый раунд шифрования включает четыре преобразования:

1. **SubBytes** — нелинейная побайтовая подстановка через таблицу S-box.
2. **ShiftRows** — циклический сдвиг строк матрицы состояния.
3. **MixColumns** — линейное преобразование столбцов с помощью умножения в поле GF(2⁸).
4. **AddRoundKey** — побитовое сложение (XOR) состояния с раундовым ключом.

В финальном раунде преобразование MixColumns не выполняется. При расшифровании применяются обратные операции в обратном порядке.

### Расширение ключа (Key Expansion)

Из исходного ключа формируется набор раундовых ключей с помощью операций RotWord, SubWord, XOR и раундовых констант Rcon. Количество раундовых ключей на единицу больше числа раундов.

### Режимы работы

- **ECB** — каждый блок шифруется независимо. Одинаковые блоки открытого текста дают одинаковые блоки шифртекста.
- **CBC** — перед шифрованием каждый блок XOR-ится с предыдущим блоком шифртекста (или с вектором инициализации IV для первого блока). Это устраняет повторяющиеся паттерны.

│   ├── block.py                # Шифрование и расшифрование одного блока
│   ├── config.py               # Параметры вариантов AES (Nk, Nr, Nb)

│   ├── files.py                # Чтение и запись файлов, работа с IV

├── tests/                      # Тесты
│   ├── selfcheck.py            # Проверка блочного шифрования по эталонным векторам

## Требования

- Python 3.10+
- Сторонние зависимости не требуются

## Использование

### Шифрование файла

**Режим ECB:**

```bash
python main.py encrypt \
  --input aes_test_input.txt \
  --output encrypted.bin \
  --mode ecb \
  --aes 128 \
  --key 000102030405060708090a0b0c0d0e0f
```

**Режим CBC с автоматической генерацией IV:**

```bash
python main.py encrypt \
  --input aes_test_input.txt \
  --output encrypted.bin \
  --mode cbc \
  --aes 256 \
  --key 000102030405060708090a0b0c0d0e0f101112131415161718191a1b1c1d1e1f
```

**Режим CBC с указанием IV вручную:**

```bash
python main.py encrypt \
  --input aes_test_input.txt \
  --output encrypted.bin \
  --mode cbc \
  --aes 128 \
  --key 000102030405060708090a0b0c0d0e0f \
  --iv 00112233445566778899aabbccddeeff
```

**Загрузка ключа из файла:**

```bash
python main.py encrypt \
  --input aes_test_input.txt \
  --output encrypted.bin \
  --mode ecb \
  --aes 256 \
  --key-file secret.key
```

Файл ключа должен содержать ключ в шестнадцатеричном формате. Параметры `--key` и `--key-file` являются взаимоисключающими.

### Расшифрование файла

**Режим ECB:**

```bash
python main.py decrypt \
  --input encrypted.bin \
  --output decrypted.txt \
  --mode ecb \
  --aes 128 \
  --key 000102030405060708090a0b0c0d0e0f
```

**Режим CBC (IV читается из начала файла):**

```bash
python main.py decrypt \
  --input encrypted.bin \
  --output decrypted.txt \
  --mode cbc \
  --aes 256 \
  --key 000102030405060708090a0b0c0d0e0f101112131415161718191a1b1c1d1e1f
```

**Режим CBC с ключом из файла и указанием IV:**

```bash
python main.py decrypt \
  --input encrypted.bin \
  --output decrypted.txt \
  --mode cbc \
  --aes 256 \
  --key-file secret.key \
  --iv 00112233445566778899aabbccddeeff
```

### Генерация таблиц S-box, Inv-S-Box и Rcon

```bash
python export_tables.py
```

Результат сохраняется в файл `generated_aes_tables.txt` и включает сгенерированные таблицы S-box, Inv-S-box и Rcon с проверкой их соответствия эталонным значениям.
